Operational control

Built For Known Devices, Not Open Relay Exposure

Relaywright focuses on the quiet work system engineers need from an internal SMTP relay: accept only what should be accepted, keep accepted messages durable, and make delivery state visible.

Trusted Device Intake

Allow SMTP submission by IP or CIDR, with per-device owner, location, sender, recipient, size, and rate policy summaries in the admin UI.

Durable Queue

Accepted DATA is written to disk before SMTP success. Operators can inspect age, attempts, retry schedule, expiry, failure category, and delivery history.

Upstream Delivery

Relay to one upstream smart host with TLS settings, optional basic authentication, or Microsoft OAuth client credentials.

Diagnostics

Run connectivity checks, submission-flow previews, and test email diagnostics without changing relay behavior or consuming production message flow.

Backups And Restore Staging

Create SQLite backups, validate restore readiness, review retention, and stage restores with clear restart impact before they apply.

Security Visibility

Review admin sign-in events, current session details, suspicious login warnings, HTTPS certificate status, and account recovery guidance.

Scope

What Relaywright Is

  • Self-hosted SMTP intake for trusted internal devices and apps.
  • A durable spool plus queue metadata before accepted clients get success.
  • An authenticated admin console for settings, logs, queue review, backups, alerts, diagnostics, and certificates.

Boundaries

What It Is Not

  • Not a public MX, mailbox server, spam filter, mailing list tool, or general-purpose MTA.
  • Not an open relay; trusted-network checks happen before SMTP submission is accepted.
  • Not an automatic updater; installers and release artifacts remain the update mechanism.

Validated release line

Windows And Linux Packages

Release builds are self-contained. The target machine does not need a separate .NET runtime.

Windows x64

Installer-based service setup with HTTPS-first admin defaults, optional scoped Windows Firewall rules, and side-by-side release updates.

Linux x64

Installer script for systemd hosts with checksum verification, service management, optional firewall configuration, and retained release folders.

Linux ARM64

Validated ARM64 package for small-office ARM devices and modern Raspberry Pi class systems running a 64-bit OS.

Linux ARMv7

Best-effort 32-bit ARM package is published until dedicated ARMv7 validation hardware is available.
