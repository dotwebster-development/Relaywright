## Deviations And Fixes

- RC6 failed the trusted SMTP smoke because mailbox address formatting used `IMailbox.ToString()`. Fixed in PR #3 by formatting `IMailbox.User` and `IMailbox.Host` explicitly and by removing the masking fake `ToString()` override from tests.
- The first SMTP smoke workflow treated a blocked TCP path as acceptable for untrusted testing. That was safe but weaker than the release gate. PR #4 changed the workflow to prove app-level rejection by temporarily allowing the exact `dot-linux01` source IP through the firewall.
- PR #5 reset per-job smoke artifacts so self-hosted runner workspace leftovers cannot contaminate release evidence.
- PR #6 added the Linux soak workflow and PR #7 fixed capture artifact counting after the first shakedown exposed filename reuse across capture-server restarts. The first run had `97` delivered messages and zero queue failures, but the harness only counted `64` capture files because earlier files were overwritten after the planned upstream restart.
- PRs #8 through #15 added and hardened Linux ugly-path validation. Early shakedown failures were harness issues around startup timing, shallow public `/health` expectations, and fake capture SMTP lifetime during service restarts. The final workflow validates safety through SMTP/queue state and uses a restart probe that first proves stale messages recover to retry state, then proves delivery after the upstream returns.
- `dot-linux02` needed managed firewall preparation before Linux validation; `ufw` is now active for the install-validation lane.
- No local or GitHub Release artifact/tag for `0.1.0-beta.1` was available during this pass. The automated upgrade coverage uses the in-repo beta-shaped fixture rather than a database generated from the exact historical release.
